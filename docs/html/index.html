add_reference_from_action_to_action(from_action_uuid, from_property_name, to_action_uuid, to_weaviate='localhost')¶

Allows to link two actions unidirectionally

Parameters

  • from_action_uuid (str in the form of an UUID) – The action that should have the reference as part of its properties. Accepts a plane UUID or an URL. E.g. ‘weaviate://localhost/actions/fc7eb129-f138-457f-b727-1b29db191a67’ or ‘fc7eb129-f138-457f-b727-1b29db191a67’

  • from_property_name (str) – The name of the property within the action.

  • to_action_uuid (str) – The UUID of the action that should be referenced. Accepts a plane UUID or an URL. E.g. ‘weaviate://localhost/actions/fc7eb129-f138-457f-b727-1b29db191a67’ or ‘fc7eb129-f138-457f-b727-1b29db191a67’

  • to_weaviate (str) – Specifies the weaviate instance on which the cross referenced action is located. Defaults to ‘localhost’. If ‘to_action_uuid’ specifies an URL then ‘to_weviate’ must match the given domain or be explicitly set to None.

Returns

None if successful.

Raises

ConnectionError: if the network connection to weaviate fails. UnexpectedStatusCodeException: if weaviate reports a none OK status. TypeError: If the parameters are of the wrong type ValueError: If the parameters are of the wrong value
