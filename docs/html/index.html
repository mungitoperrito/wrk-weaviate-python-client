add_reference(from_uuid, from_property_name, to_uuid, from_semantic_type='things', to_semantic_type='things', to_weaviate='localhost')¶

Allows to link an entity from a thing unidirectionally.

Parameters

  • from_uuid (str in the form of an UUID, str in form of URL) – The entity that should have the reference as part of its properties. Accepts a plane UUID or an URL. E.g. ‘weaviate://localhost/things/fc7eb129-f138-457f-b727-1b29db191a67’ or ‘fc7eb129-f138-457f-b727-1b29db191a67’ By default the entity is a thing please specify from_semantic_type if you want to reference from an action.

  • from_property_name (str) – The name of the property within the entity.

  • to_uuid (str in the form of an UUID, str in form of URL) – The UUID of the entity that should be referenced. Accepts a plane UUID or an URL. E.g. ‘weaviate://localhost/things/fc7eb129-f138-457f-b727-1b29db191a67’ or ‘fc7eb129-f138-457f-b727-1b29db191a67’

  • from_semantic_type (str) – Either things or actions. Defaults to things. Settable through the constants SEMANTIC_TYPE_THINGS and SEMANTIC_TYPE_ACTIONS

  • to_semantic_type (str) – Either things or actions. Defaults to things. Settable through the constants SEMANTIC_TYPE_THINGS and SEMANTIC_TYPE_ACTIONS

  • to_weaviate (str) – Specifies the weaviate instance on which the cross referenced thing is located. Defaults to ‘localhost’. If ‘to_thing_uuid’ specifies an URL then ‘to_weviate’ must match the given domain or be explicitly set to None.

Returns

None if successful.

Raises

ConnectionError: if the network connection to weaviate fails. UnexpectedStatusCodeException: if weaviate reports a none OK status. TypeError: If the parameters are of the wrong type ValueError: If the parameters are of the wrong value
