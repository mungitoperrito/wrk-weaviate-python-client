add_reference_to_thing(from_thing_uuid, from_property_name, to_thing_uuid, to_weaviate='localhost')¶

Allows to link two objects unidirectionally.

Parameters

  • from_thing_uuid (str in the form of an UUID) – The thing that should have the reference as part of its properties. Accepts a plane UUID or an URL. E.g. ‘weaviate://localhost/things/fc7eb129-f138-457f-b727-1b29db191a67’ or ‘fc7eb129-f138-457f-b727-1b29db191a67’

  • from_property_name (str) – The name of the property within the thing.

  • to_thing_uuid (str in the form of an UUID) – The UUID of the thing that should be referenced. Accepts a plane UUID or an URL. E.g. ‘weaviate://localhost/things/fc7eb129-f138-457f-b727-1b29db191a67’ or ‘fc7eb129-f138-457f-b727-1b29db191a67’

  • to_weaviate (str) – Specifies the weaviate instance on which the cross referenced thing is located. Defaults to ‘localhost’. If ‘to_thing_uuid’ specifies an URL then ‘to_weviate’ must match the given domain or be explicitly set to None.

Returns

None if successful.

Raises

ConnectionError: if the network connection to weaviate fails. UnexpectedStatusCodeException: if weaviate reports a none OK status. TypeError: If the parameters are of the wrong type ValueError: If the parameters are of the wrong value
